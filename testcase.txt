test case

Это документ с описанными четкими действиями, которые нужно выполнить, чтобы проверить функцию продукта. Эти действия направлены на проверку того, что функция работает должным образом и соответствует стандартам и требованиям клиента. Это проверка получения ожидаемого результата.
Если несколько test case проверяют одну функциональность, от их можно объединять в тест-наборы test set

Атрибуты

1. ID - номер (обязательный). Может создаваться автоматически системой управления тестированием.
2. Заголовок - название (обязательный). Отражает суть, описывает что будет проверяться. Должен быть понятным и коротким.
3. Предусловия (необязательный). Указываются действия, которые необходимо выполнить перед выполнением основных действияий. Не имеют прямого отношения к проверке. Могут содержать тестовые данные, если они не вынесены в отдельный документ.
4. Шаги (обязательный). Описываются последовательность действий, которые необходимы для проверки. Нумеруются. Если шаг один, то его можно не нумеровать. Действия должны быть описаны понятно. Должны использоваться безличные глаголы(перейти, открыть, нажать).
5. Постусловия (необязательный). Указываются действия не связанные с основной проверкой, но необходимые для возврата системы в исходное состояние.
6. Ожидаемый результат (обязательный). Описывает результат, который ожидается получить в результате успешно выполненных действий. Может быть общим для всего test case.
7. Требования к среде (необязательный). Указание оборудования, программного обеспечения и другого необходимого для прохождения.
8. История редактирования (необязательный). Журнал кто, когда и почему вносил иззменения.
